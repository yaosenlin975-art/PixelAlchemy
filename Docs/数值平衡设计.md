## 0. 开放决策数值落点建议（D1–D5）

> 配合策划（谋远）的设计方向给出数值落点。下游（协议/相机/HUD/结算）依赖这些拍板。建议主理人写入 ADR。

| 决策 | 数值侧建议 | 一句话理由 |
|------|-----------|-----------|
| **D1 边界规则** | **采用 `Kill`**；配「击退冲量」模型把人挤出去（详见 §4） | 招牌机制；出界即死最干净、最对称、最易平衡 |
| **D2 缩圈** | **v1 不做缩圈**（MVP 优先）；BR 模式再用「移动 kill-ring」变体（§5） | 4 人 256px 乱斗靠边界博弈已收口，缩圈徒增网络/相机负荷 |
| **D3 Dash** | **保留**：距离 34px、CD 1.8s、可被固体阻挡、无资源消耗、**MVP 无 i-frames（纯位移）**（§3.3） | 移动端逃生公平需要；是边界博弈的「交互性」而非「苟」 |
| **D4 HP** | **v1 纯出界致死（无 HP）**；HP 仅作可选配置（§6）。GDD Q1 一致：环境只逼走位不致死 | 保持「百战天虫」身份，无血条 HUD、无数值膨胀 |
| **D5 AimPower/切换** | **v1 固定力度**（不加字段，预留 uint8）；**法术循环切换、4 槽、免费、不消耗回合**；施法受 **Mana + CD 双闸**（§7） | 避免协议 bump；移动友好；Mana 是「推出去」节奏唯一门控 |

### D1 — Kill 边界 + "挤出去"推人模型（核心）
- **致死判定**：玩家中心 `cx ∉ [-16, W+16]` 或 `cy > H+16` → 立即 `PlayerDeath`。边界余量 16px（对齐 GDD §4.2）。
- **推人靠「击退冲量」**：法术/爆炸/反应对玩家速度叠加冲量 `K`（px/s），按 `v *= 0.90/step` 衰减，总位移 ≈ `K / 3.0` px。
- 初版量级：轻弹 45（≈15px）、重弹 90（≈30px）、爆炸 150（≈50px，半径 40px 径向衰减）、熔岩接触 30/0.3s、自身爆炸 ×0.5（技能：别站自己爆点）。详见 §4 表。
- **设计铁律**：只有「在边缘 15–55px 内被命中」才会被推出界 → 边缘控制是技术点，轻弹乱撸推不出去，杜绝无脑 push。

### D2 — 缩圈（仅建议，v1 不做）
- 若做，用 **kill-ring 变体**（移动致死边界，无需 HP）：4 阶段，5s 预警 + 25s 收缩 + 30s 维持；安全半径 `R→0.75R/阶段`；圈外=即死（同边界）。
- 伤害-ring 变体需 HP（§6），本报告不推荐 v1。

### D3 — Dash（保留，MVP 无 i-frames，纯位移）
- `Distance 34px`（范围 28–42）、`Duration 0.16s`（≈5 帧，隐含速度 ≈210px/s）、`Cooldown 1.8s`（1.5–2.5）。
- **可被固体阻挡**：Stone/Wood/Ice 命中即停（不穿墙）。气体/液体/粉末不阻挡。
- **无资源消耗**（仅 CD 门控）；Dash 期间 **仍可被击退推出界**（Dash≠保命，只是水平位移）。
- **MVP 无 i-frames（纯位移）**：Dash 为纯水平位移，**不免疫任何效果（含环境控制/减速/致盲/材料 debuff）**，期间仍可被击退推出界。对齐 ADR D3 / GDD §6.1.1（六份文档一致）。
- 若未来开 HP 模式（D4-B）：Dash **仍保持纯位移、不加 i-frames**（避免 dash 成为硬逃工具）；免伤由 Shield 等独立机制处理。

### D4 — HP（v1 无；可选）
- 纯 Kill 模式下无 HP、无血条。若开启 HP 配置：`BaseHP=100`，法术直伤 8–38、环境 DoT 4–16/s，HP 模式 TTK 20–40s；**出界仍即死**（双死因并存）。环境危害主职仍是逼走位（对齐 GDD Q1）。

### D5 — AimPower + 法术切换 + Mana
- **AimPower**：v1 固定力度（每法术定义固定弹速/射程），**不加 InputPayload 字段**；预留 `uint8 ×100 [0,100]`。若后期开启：蓄力 `Tmax=1.2s`，`p=(t/Tmax)^0.8`，**只缩放射程+弹速、不缩放伤害**（防蓄力秒杀）。
- **法术切换**：循环切换（SelectNext/Prev），4 槽，**免费、不消耗回合/动作**；施法受 **Mana + 每法术 CD 双闸**门控（切换自由但放出有代价）。对齐 input-arch Q6。
- **Mana（MVP 双闸）**：Mana 是仿真态（非 InputPayload 字段），加 Mana 不破坏 D5 wire 稳定。v1 即启用，与每法术 CD 共同约束击退频率，避免 4 个低价法术轮切连续推人（详见 §6.3）。

## 1. 数值框架总览

### 1.1 单位与确定性约束（P0）
| 量 | 单位 | 说明 |
|----|------|------|
| 位置 | px（整数网格） | 像素坐标；玩家中心 = 盒中心 |
| 速度 | px/s | 所有位移基于 `v*dt`，dt=1/30 |
| 角度 | AimAngle int16 [0,628] | ×100 rad，0.01rad 分辨率 |
| 时间 | 逻辑帧 | 30Hz；动画/渲染 60Hz 插值 |
| 数值类型 | Fix64 | 全仿真无 float；Rand 用确定性 PRNG（seed 下发） |

### 1.2 属性维度（PlayerStats，P0）
玩家状态由以下确定性字段构成（进快照、可回滚）：
`PosX, PosY, VelX, VelY, AimAngle, SelectedSpellSlot, Mana, DashCD, CastCD[], (HP 可选), ActionFlags`。

### 1.3 资源循环（P0）
```
        ┌─────────── 法术施放(双闸: Mana + 每法术CD) ───────────┐
        │ 消耗 Mana │  产生 场上材料/弹体                        │
        ▼            ▼                                          │
   Mana 回复 ◄── 冷却      材料/反应 ◄──────────────────────────┘
                        │
                  反应(水+火→蒸汽…)
                        │
                  Kill 向量: 击退冲量 / 地形改造
                        │
                  出界即死 / (HP 归零)
```
- **Mana** 是「推出去」节奏的唯一门控：没 mana 就放不出能把人推走的法术 → 击退频率受限，避免秒推。仅每法术 CD 时，4 个低价法术可轮切连续推；故 v1 用 Mana + CD 双闸。
- **材料** 不另设货币：来自地形（挖/落）与法术召唤（火/冰/毒/酸/水/熔岩/石为召唤材料）。道具（GDD §7）是独立公平争夺的增益，非原材料。

## 2. 移动 / 跳跃 / Dash 初版量级（P0）

| 属性 | 默认 | 范围 | 单位 | 备注 |
|------|------|------|------|------|
| `MoveSpeed` | 75 | 65–90 | px/s | 输入向量模长≤100 归一成全速（对角不加速，公平） |
| `Gravity` | 220 | 180–260 | px/s² | 每步 `vy -= 220/30` |
| `JumpVelocity` | 110 | 100–125 | px/s | 跳高≈27px，滞空≈1.0s；v1 单跳 |
| `AirJumps` | 0 | — | — | P1 可考虑二段跳 |
| `PlayerBox` | 4×6 | — | px | w×h；中心为出界判定基准 |
| `MaxSpeed`(cap) | 320 | 280–360 | px/s | 反作弊位移上限；每步容忍 `×1.5` = 16px |

**跨步校验**：合法单步位移 ≤ `MaxSpeed*dt*1.5` = 320/30*1.5 ≈ 16px。Dash(≈7px/步)、击退(≤5px/步)均在其内。

### 2.1 Dash（D3，P0，MVP 无 i-frames，纯位移）

## 3. 边界与死亡模型（D1，P0）

### 3.1 击退冲量模型
- 命中时对玩家速度叠加 `K_dir * K_mag`（px/s）。
- 衰减：`v_knock *= 0.90` 每步（dragCoef=0.10/步，时间常数≈0.33s）。
- 近似总位移：`disp ≈ K_mag / 3.0`（px）。例如 K=150 → ≈50px。

### 3.2 击退量级表（初版，设计铁律：仅边缘 15–55px 内被命中才出界）
| 来源 | K_mag(px/s) | 近似位移 | 半径/方式 | 备注 |
|------|------------|---------|----------|------|
| 轻弹（基础 Bolt） | 45 | 15px | 点命中 | 仅边缘≤15px 危险 |
| 重弹（Heavy） | 90 | 30px | 点命中 | 边缘≤30px |
| 爆炸（Bomb/火+毒） | 150 | 50px | 40px 径向，×(1−d/R) | 主推人向量 |
| 熔岩接触 | 30/0.3s | ~10px | 持续 | 小型连续推（非致死） |
| 水/液流 | 12/s 拖拽 | — | 持续 | 减速+轻推 |
| **自身爆炸** | **×0.5** | ~25px | — | 技能：勿站己爆点 |

### 3.3 致死与名次
- 中心越界 → 即时 `PlayerDeath` → 观战（1s 延迟）。
- 多人同帧出界：按出界顺序定名次（GDD §4.3）。
- **Bounce/Wrap 变体**（房主可配，非默认）：Bounce 边界反弹（不致死，强调材料战）；Wrap 对侧出现（混沌娱乐局）。数值上仅改变边界处理函数，不影响击退表。

## 6. 法术数值（D5，P0/P1）

### 6.1 Aim / AimPower
- `AimAngle`：[0,628]（0.01rad≈0.57°），固定量化。
- **v1 固定力度**：每法术内置固定弹速/射程，不加字段。
- **预留**：`AimPower uint8 ×100 [0,100]`。若开启：
  - 蓄力 `Tmax=1.2s`（1.0–1.5），`p=(t/Tmax)^0.8`。
  - **只缩放射程+弹速，不缩放伤害**（防蓄力秒杀）。
  - 量化 `uint8 ×100 [0,100]`，进 `InputPayload` 需 bump 协议版本。

### 6.2 法术切换
- 循环切换（SelectNext/Prev），**4 槽**，`SelectedSpellSlot` 为推导状态（0–3 循环）。
- **免费、不消耗回合/动作**；施法受 **Mana + 每法术 CD 双闸**门控。
- 起始 1–2 固定 + 卷轴临时学（≤4）。对齐 input-arch Q6。

### 6.3 Mana（资源，P0 — MVP 双闸）
| 参数 | 默认 | 范围 |
|------|------|------|
| MaxMana | 100 | — |
| Regen | 15/s | 12–18 |
| 轻弹耗 | 10 | 8–15 |
| 重弹耗 | 25 | 20–30 |
| 爆炸耗 | 38 | 30–45 |
| 材料召唤耗 | 12–20 | — |

> Mana 为仿真态，非 InputPayload 字段，引入不破坏 D5 wire 稳定。v1 即启用，与每法术 CD 共同约束击退频率（防止轮切连续推）。

### 6.5 Shield 护盾泡（P1，定义）
- 效果 = **3s 免疫环境 DoT + 法术直伤**（HP 模式相关）。
- **不免疫击退**：否则在 no-HP 模式下可无限抗推、直接破 Kill 边界张力。
- no-HP 模式下价值低（无 DoT/直伤可免）→ 故置 P1；若 v1 需要防御类，可降为 P0 但必须保持「不免疫击退」。

## 8. 道具数值（对齐 GDD §7，P0/P1）
| 道具 | 效果（具体） | 权重 |

## 9. 平衡原则（对称竞技公平性，P0）

1. **对称出身**：所有玩家基础属性完全相同；差异仅来自公平争夺的道具（Telegraph 3s 预告，所有人可见）。
2. **确定性即公平**：同输入→同结果，无外挂信息优势；帧同步 + 服务器裁决。
3. **击退平等**：击退冲量大小与施法者无关（自身爆炸 ×0.5，作为技能惩罚而非保护）。
4. **出身公平**：种子确定性生成，最小间距 ≥ `Width/4`。
5. **无付费优势**：MVP 无商业化，无数值膨胀通道（纯 Kill 模式天然无等级/数值成长）。
6. **先手中性**：30Hz + 输入延迟统一，两端无客户端帧优势。
7. **边界张力优先**：任何「免疫击退」类效果（Shield/Blink 若给免推、或任何免推）在 no-HP 模式下一律禁止，否则破坏 Kill 招牌机制；Dash 为纯位移、无 i-frames，天然不具免推。

## 13. 法术后坐力数值（RecoilSpeed / 自爆护栏，P0）

### 13.2 自爆护栏（anti self-out-of-bounds / 防误杀）

- **物理权威 DRAG = 0.90/步**（×0.90），原 0.85 推导作废。
- **自由空间位移**：`D = v0 · dt / (1 − r) = v0 · 0.3333`（dt=1/30，r=0.90）。
- **护栏设计封顶 RecoilSpeed_MAX_PHYS = 405 px/s**（主理人锁定，非 [TUNING]；取平衡包络上限，保证对未来 ≤405 的强后坐力法术亦成立）。
- **安全阈值 T = 405 × 0.3333 ≈ 135 px = 2.25 × bodyWidth**（bodyWidth=60 [TUNING 假设]）。
- **预警半径 N = 135 px**（= T，统一护栏半径）。
- **✅ 护栏已裁决统一（主理人 2026-07-07 终裁）**：经读盘复核，`法术系统设计.md` §3.4.9 v1.1 落盘值即 **T=135px（安全半径，非硬位移 clamp）**，并明确"90px 不再作为安全半径"；端木林上轮引用的"90px override 135"为**误读旧稿**，两份权威源实际一致（均=135）。`法术后坐力确定性落地.md` §6.1/§6.6 已同步锁定 `T=135px`、`MAX_RECOIL_SPEED=405px/s`。本 § 维持 T=135px，无误冲突。Heavy/Bomb/Fire 位移 106.7/120/130px 差异化全保留，零新哈希字段。
- **含义**：
  - 开阔地（施法者距任意边界 > 135px）：发射不因后坐力触界 → 防误杀 / 防友伤（anti friendly-fire）。
  - 贴边（距边界 ≤ 135px 且朝场内发射）：仍触发自爆，保留「近界施法有代价」的战术边界张力（保战术）。
- **与 B 修订的一致性（冲突消解）**：主理人 A 草稿按 Lava=405 得 T=135；B 将 Lava 归非发射（=0）。因非发射类无 caster 反冲，**护栏仅受发射类约束**，当前发射类封顶=Fire 390 → 实际最坏位移 `D=390×0.3333≈130px < T=135px`。护栏对全部现版本法术成立且略保守（向前兼容至 RecoilSpeed_MAX_PHYS=405）。若主理人要求严格按 A 字面「Lava 封顶=405」释义护栏，则 T 维持 135（本文取法），且 405 仅作护栏设计封顶、不直接对应某发射类现值的标注，已在此说明。

### 13.3 与物理耦合（DRAG=0.90 实值已确认）

- **DRAG=0.90/步 已是物理事实，非 [TUNING]**：与本文档 §3.1（v_knock *= 0.90/步，dragCoef=0.10，时间常数≈0.33s）、§1 顶部（v*=0.90/step，总位移≈K/3.0）完全一致。
