## 1. 模拟层字段决策

**结论：不新增独立 `RecoilVelocity` 字段。后坐力冲量在 `ApplyCastSpell` 成功后直接叠加到施法者持久速度向量 `PlayerData.VelX/VelY`，复用现有速度积分器（与 Move / Knockback 同一积分器线性叠加）。**

速度模型（精确化，对齐 `法术系统设计.md` §3.4.4 的三来源描述）：

- `PlayerData.VelX/VelY` = **持久冲量速度**（Recoil + Knockback），每 tick ×`DRAG_MULT` 衰减。
- **MoveVelocity 不持久存储、不衰减**：每帧由 `(MoveX, MoveY)` 重建为 `normalize(MoveX,MoveY) * MOVE_SPEED`，在位置积分时与持久冲量加总。可反向抵消、可瞬停，符合 §3.4.4「无（持续输入，可反向抵消）」。
- 位置积分时：`pos += (MoveVel + VelX/VelY) * dt`，随后 `VelX/VelY *= DRAG_MULT`。

为什么「并入」最优（确定性红线视角）：

1. **状态哈希只哈希一次**：`MurmurHash3`（每 60 帧比对）本来只对 `PlayerData.Position` + `VelX/VelY` 哈希。后坐力进入 `VelX/VelY` → **零新增哈希字段、零新增快照字段**。
2. **回滚零成本**：`ReplayForward` 重放 `ActionFlags.Attack` 边沿时，`ApplyCastSpell` 确定性重建后坐力，无需任何额外存档状态。
3. **若保留独立 `RecoilVelocity` 通道**（§3.4.4 允许的「如需内部分离」）：须再哈希 2 个 Fix64、进快照、且重放须精确重建该通道 → 确定性表面积翻倍、desync 风险↑。**MVP 不采用。**
4. 若未来需客户端侧单独标记「后坐力来源」用于 VFX：应为**纯渲染层推导**（由本帧速度增量或一次性事件标记），**不可**成为 `PlayerData` 字段；如确需进模拟，则必须进快照 + 哈希（红线）。

## 2. 触发实现（ApplyCastSpell 内）

**触发点**：`ActionFlags.Attack` **边沿** → 模拟走 `CastSpell` 流程（读 `slots[SelectedSpellSlot]`、方向 = `AimAngle`）→ **施法成功（Mana + 每法术 CD 双闸通过）后** → 施加后坐力。

伪代码（Fix64，禁用任何浮点）：

```csharp
// 在 ApplyCastSpell 施法成功分支内、生成投射物之后
Fix64 speed = RECOIL_TABLE[spell.SpellId];   // px/s，确定性只读常量表（终值见 §6.2）
speed = Fix64Math.Min(speed, MAX_RECOIL_SPEED);  // 设计上界 clamp：MAX_RECOIL_SPEED=405px/s(135/0.3333)=RecoilSpeed_MAX_PHYS；当前最大 Fire 390<405 不触发，安全半径 T=135px 非硬位移 clamp（见 §6.1）
if (speed != Fix64.Zero)
{
    // AimAngle 为 int16 [0,628]，×100 表示 [0, 2π)
    Fix64 aimRad = Fix64Math.Atan2(dirY, dirX);   // 或查预量化表，禁用 Math.Atan2
    Fix64 dirX = Fix64Math.Cos(aimRad);           // 必须用 Fix64Math，禁用 Math.Cos/Sin
    Fix64 dirY = Fix64Math.Sin(aimRad);
    // 反冲方向 = -aimDir（即 AimAngle + π）
    player.VelX += -dirX * speed;   // 进入持久冲量向量，随后续积分器衰减/碰撞
    player.VelY += -dirY * speed;
}
ClampTotalSpeed(player, V_MAX);     // V_MAX = 1000 px/s（见 §6.4）
// 之后统一走：drag → 位置积分 → 墙体碰撞/滑移 → 边界判定（与 Move/Knockback 同一积分器）
```

数据来源：`RECOIL_TABLE` 是法术元数据表（`SpellMeta`）的一个字段（与初速 / 冷却 / 耗蓝同表），**确定性只读、不依赖 PRNG / 随机**。键 = `spell.SpellId`（由 `SelectedSpellSlot` 推导）。

同帧 `Dash` 位（ADR D3）：按 `法术系统设计.md` §3.4.6 先执行 Dash 位移、再叠加后坐力（或同帧合成），最终位置同时反映两者；二者都无 i-frames、都不免疫效果。

## 3. 确定性保证（红线）

后坐力 = `f(SelectedSpellSlot 推导法术, AimAngle, RECOIL_TABLE 常量)`，全部满足：

- **Fix64 定点运算**：方向用 `Fix64Math.Cos/Sin`，**禁用 `Math.Cos/Sin` 浮点发散**（跨平台一致性关键）。
- **无 PRNG / 无浮点 / 无平台差异 / 无时间依赖**。
- **不进 `InputPayload`**（8B 不变），由模拟在 `ApplyCastSpell` 内派生。

**回滚**：`SelectedSpellSlot` 由 `SelectNext/Prev` 位确定性推导并进快照，重放 `Attack` 边沿即重建，无额外状态。

**参与状态哈希的字段（后坐力相关子集，MurmurHash3 每 60 帧比对）**：

| 字段 | 类型 | 后坐力是否修改 |
|------|------|----------------|
| `PlayerData.Position` (X, Y) | Fix64 | 间接受影响（速度改变 → 位移改变） |
| `PlayerData.VelX, VelY` | Fix64 | **直接修改（唯一新增量落点）** |
| `PlayerData.SelectedSpellSlot` | int (0–3) | 否（推导法术来源，已哈希） |
| `PlayerData.SpellIds[4]` | loadout | 否（推导法术来源，已哈希） |
| `PlayerData.Mana` / 每法术 CD 计时器 / alive 标志 | — | 否（施法闸已哈希） |

> 完整集合以 `多人联机帧同步对战设计.md` §9.2 `PlayerState` 快照为准。后坐力**不向哈希引入任何新字段**。

## 4. 边界 / 反弹 / 阻力

- **复用现有边界检测**：Kill 模式（ADR D1 默认）出界即死 → 后坐力推出界 = 自爆，自然成立、**允许**（`法术系统设计.md` §3.4.5）。
- **后坐力速度不豁免** drag / 反弹 / 墙体碰撞：与 Move / Knockback 同积分器、同衰减。
- **阻力**：每 30Hz tick 速度 ×`DRAG_MULT`。已拍板：`DRAG_MULT = 0.90/步`（×0.90，NUMERIC §3.1），详见 §6.1（brief 的 0.85 作废）。
- **墙体碰撞**：撞 Stone / Wood / Ice 固体阻挡 / 滑移（不穿墙），与 Dash 一致；气体 / 液体 / 粉末不阻挡。
- **Bounce 模式**（房主可选，非默认）：撞边界按 `restitution`(<1) 反射、消耗能量；`restitution` 数值由物理层定（见 §6.3 待定项）。

## 5. 60Hz 渲染插值

后坐力是速度，随 30Hz 帧在 60Hz 渲染间线性插值，**无需为后坐力本身加 Telegraph**。若需「发射预警」视觉，由 TA / 客户端处理、不进模拟、不进哈希。

## 6. 物理常量与 RECOIL_TABLE（A / B 已拍板）

### 6.1 阻力系数 DRAG_MULT（已拍板）与位移护栏（T=135px 安全半径，已锁定）

**DRAG_MULT（A 拍板）**：采用物理层权威值 `DRAG_MULT = 0.90/步`（×0.90）；brief 的 0.85 作废。后坐力按 `法术系统设计.md` §3.4.4「同落沙阻力衰减」与击退同系数（`数值平衡设计.md` §3.1：`v_knock *= 0.90 每步`，`disp ≈ K_mag/3.0` 与此一致）。

**自由空间位移（未 clamp）**：`D = v0·dt / (1 − r) = v0 · (1/30) / 0.10 = v0 · 0.3333`（dt=1/30，r=0.90，显式 Euler：先 `pos += v·dt` 再 `v ×= r`）。

**位移护栏 = 安全半径 `T=135px`（已锁定，非硬位移 clamp）**：对齐 `法术系统设计.md` §3.4.9（主理人 2026-07-07 锁定）。`T=135px`（=2.25×bodyWidth，bodyWidth=60 为 [TUNING] 假设）为"安全半径"——开阔地距边界 >135px 施法不会自爆；**非对自由空间位移做硬截断**（位移按 `D = v0·0.3333` 自然衰减，Heavy/Bomb/Fire = 106.7/120/130px，均 <135px，差异化保留）。`ApplyCastSpell` 内 `speed` 仍 clamp 到 `MAX_RECOIL_SPEED=405px/s`（=135/0.3333，见 §2 伪代码），该值为设计上界 `RecoilSpeed_MAX_PHYS`，当前最大 Fire 390<405 不触发，零新哈希字段（结果只进 `VelX/VelY`）。

| 法术（实际有后坐力） | RecoilSpeed (px/s) | D 自由空间 (px) | 实际位移 (px，安全半径 T=135px 不截断) |
|------|--------------------|----------------|----------------|
| Bolt | 120 | 40.0 | 40.0 |
| Water | 160 | 53.3 | 53.3 |
| Ice | 220 | 73.3 | 73.3 |
| Poison | 240 | 80.0 | 80.0 |
| Acid | 260 | 86.7 | 86.7 |
| Heavy | 320 | 106.7 | 106.7 |
| Bomb | 360 | 120.0 | 120.0 |
| Fire | 390 | 130.0 | 130.0 |

（StoneWall / Lava / Shield / Blink 实际后坐力 = 0，见 §6.2；Lava 设计量级 405 已作废，不参与位移。）

**护栏效果（玩法侧验收，对齐 SPELL §3.4.9 T=135px）**：
- **开阔地**（距边界 > 135px）放任意法术：反冲位移 ≤ 130px（Fire 最坏）< 135px → **不会被自己推出界**（消除「平地误杀」）。
- **贴边**（距界 ≤ 135px）**朝场内发射**：反冲朝外、位移足以出界 → **自爆风险真实保留**（战术张力成立）。
- 即「贴边自爆」为有意保留张力，「平地误杀」为被安全半径消除的意外。

> **护栏裁定（主理人 2026-07-07 终裁）**：经读盘复核，**SPELL §3.4.9 v1.1 落盘值即 T=135px（安全半径，非硬位移 clamp），并明确"90px 不再作为安全半径"**；端木林上轮引用的"谋远硬 clamp 90px override 135"为**误读旧稿**，谋远从未在落盘版 override 135，两份权威源实际一致（均=135）。故裁定：**锁定 T=135px 安全半径模型**（= A 拍板 = 舒衡 NUMERIC §13.2 = GDD §6.2.1 = ADR D6）。
> - `MAX_RECOIL_DISPLACEMENT` 改为语义 **安全半径 T=135px（非硬位移 clamp）**；`MAX_RECOIL_SPEED=405px/s`（=135/0.3333，设计上界 RecoilSpeed_MAX_PHYS），当前最大 Fire 390<405 不触发，零新哈希字段。
> - Heavy/Bomb/Fire 位移 **106.7/120/130px** 差异化全保留，符合用户"不同法术后坐力"核心诉求。
> - 90px 硬 clamp 方案作废：会将三者压成统一 90px（破坏差异化手感），且与 SPELL §3.4.9"非硬位移 clamp"冲突。

### 6.2 StoneWall / Lava 后坐力（已拍板 = 0）

**拍板结论（B）**：采用 `法术系统设计.md` §3.4.7 权威分类——StoneWall（定点召唤 / 地形生成）、Lava（地面喷发）为非发射 / 喷发类，**后坐力 = 0**。

**最终 RECOIL_TABLE（实际应用于模拟，键 = `spell.SpellId`，单位 px/s，0 = 无后坐力）**：

| 法术 | 发射类? | 后坐力 (px/s) | 备注 |
|------|---------|---------------|------|
| Bolt | 是 | 120 | 轻弹，反冲弱 |
| Water | 是（持续锥） | 160 | 持续小反冲 |
| Ice | 是 | 220 | 高速弹 |
| Poison | 是 | 240 | 抛物 |
| Acid | 是 | 260 | 直线/锥 |
| Heavy | 是 | 320 | 重型主反冲源 |
| Bomb | 是（投掷） | 360 | 重型 |
| Fire | 是 | 390 | 重型 |
| StoneWall | 否（地形生成） | **0** | 原 brief 200 → 0 |
| Lava | 否（喷发） | **0** | 原 brief 405 → 0；设计量级 405 仅作护栏上界（见 §6.1） |
| Shield | 否（自环绕） | 0 | 防御 |
| Blink | 否（瞬移） | 0 | 位移 |

- 重型有后坐力法术 = **Heavy / Bomb / Fire（320 / 360 / 390）**，与舒衡表对齐。
- 实现：`RECOIL_TABLE` 为「按 `spell.SpellId` 查表、`0` 表示无后坐力」的数据驱动结构（`SpellMeta` 字段）。**代码只改表、不改逻辑**；若未来需给喷发类微量反冲，改表值即可，无需协议 / 逻辑变更。

### 6.3 反弹系数 restitution（[TUNING] 待物理层补）

Bounce 模式（非默认）边界反弹系数 `restitution`(<1) 数值「由物理层定」，当前文档未给具体值 → **物理层（mouyuan）补常量并标 `[TUNING]`** 进入确定性常量区（§6.6）。

### 6.4 V_MAX 总速度 clamp

brief 推荐 `V_MAX = 1000 px/s` 总速度 clamp。后坐力单独（≤390 实际）远低于此；叠加 Move + Knockback 可能触顶。clamp 为速度纯函数、确定性安全，进入确定性常量区（§6.6）。

### 6.5 积分顺序（建议，待物理层确认现有实现）

每 30Hz tick 固定顺序（显式 Euler，与 §3.1 `disp≈K/3.0` 一致，D 公式见 §6.1）：

1. 施加本帧冲量（Move 意图重建、Knockback、Recoil）——均在 `Apply*` 内完成；
2. `V_MAX` clamp；
3. drag：`vel × DRAG_MULT`（0.90）；
4. 位置积分：`pos += vel × dt`；
5. 墙体碰撞 / 滑移；
6. 边界判定（Kill = 出界即死 / Bounce = 反射）。

若现有积分器顺序不同，须物理层确认并对齐公式（尤其 drag 与位置积分的先后，影响 closed-form 位移常数项）。
