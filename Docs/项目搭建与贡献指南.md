## 4. 协议导出流程（关键）

> 协议以 `Tools/NetworkProtocol/*.proto` 为权威源，由 `ProtocolExportTool` 生成到服务端 Entity 与客户端 Hotfix 的 `Generate/` 目录。**禁止手改 Generate 目录**。

### 4.1 步骤

```bash
# 1. 编辑 Tools/NetworkProtocol/Outer/OuterMessage.proto（追加消息，保留字段号）
# 2. 运行导出工具
cd Tools/ProtocolExportTool
dotnet run -- --config ExporterSettings.json
# 3. 确认产物：
#    服务端: Server/Entity/Generate/NetworkProtocol/*.cs
#    客户端: Client/Assets/Scripts/Hotfix/Generate/NetworkProtocol/*.cs
```

### 4.2 协议双层序列化约定

| 用途 | 序列化 | 例子 |
|------|--------|------|
| 高频小包（输入/帧批次/快照分片/时钟/观战） | MemoryPack | `C2G_Input` / `G2C_FrameBatch` |
| 低频一次性 / 路由消息（聊天/匹配/房间/重连/场景事件） | protobuf | `G2C_SceneEvent` / `C2G_ReconnectRequest` |

### 4.3 ⚠️ ExporterSettings.json 路径陷阱

```json
{
  "Export": {
    "NetworkProtocolDirectory": {
      "Value": "d:/Unity/Projects/Noita/Tools/NetworkProtocol"
    },
    "NetworkProtocolServerDirectory": {
      "Value": "d:/Unity/Projects/Noita/Server/Entity/Generate/NetworkProtocol"
    },
    "NetworkProtocolClientDirectory": {
      "Value": "d:/Unity/Projects/Noita/Client/Assets/Scripts/Hotfix/Generate/NetworkProtocol"
    }
  }
}
```

> ⚠️ 必须为 **Windows 绝对路径**（正斜杠 `/` 兼容）。原默认值 `/Users/fantasy/...` 是 macOS 路径，会导致导出失败。若在本机换盘符，请同步修改此处三处。

### 4.4 服务端生成目录编译项

`Server/Entity/Server.csproj`（或 Entity.csproj）需包含：

```xml
<Compile Include="Generate/NetworkProtocol/**/*.cs" />
```

## 5. 程序集边界（AOT vs Hotfix）

对齐架构文档 §4.1、§4.3、GAMEPLAY §3：

| 代码类型 | 程序集 | 热更 | 说明 |
|---------|--------|:---:|------|
| DOTS ISystem / IJobChunk / 组件 | `NoitaCA.Core / Simulation / Renderer / Gameplay / Editor` | ❌ AOT | Burst 编译，确定性模拟 |
| 帧同步调度 | `NoitaCA.Lockstep` | ❌ AOT | 绝不在 Hotfix |
| 协议 handler / UI / 匹配请求 | `Hotfix.asmdef` | ✅ | 仅 handler/UI/config，禁写 ECS |
| 组合根 Bootstrap | `Assembly-CSharp`（AOT 目录） | ❌ | 4 个 MB（WorldBuilder/CameraRig/ArtDirector/GameBootstrap） |

**Hotfix 禁令**（架构文档 §4.1）：
- ❌ 访问 `EntityManager` / 写 `IComponentData` / 跑 `IJobChunk`
- ❌ Battle 状态下热更（会 desync）

## 7. ⚠️ NuGet 版本锁定（待修项）

架构文档 §2.3 第 3 项：服务端 `Fantasy-Net` / `NLog` 包版本曾用 `*`，**必须锁定具体版本**，避免还原时意外升级破坏确定性 / 接口。

```xml
<!-- Server/Entity/Server.csproj 示例（版本号以实际验证为准） -->
<PackageReference Include="Fantasy.Net" Version="2025.2.1402" />
<PackageReference Include="NLog" Version="5.3.x" />
```

> ✅ 检查清单（`开发智能体配置.md §9`）：全局搜索未用 `*` 作为包版本。

## 8. 服务端运行（.NET 8）

```bash
cd Server/Main
dotnet run --project Main.csproj
# 或发布后运行：
dotnet publish -c Release -o ./bin/publish
./bin/publish/Main
```

服务端三层职责（架构文档 §5.1）：Main（网络/Session/NLog）→ Entity（房间/玩家/哈希/反作弊 AOT）→ Hotfix（匹配/房间规则/反作弊参数/场景时间线）。

## 9. 双语注释与编码约定（速记）

完整版见 `开发智能体配置.md §4`。要点：

- **双语注释成对**：`// 职责：` + `// Responsibility:` 覆盖所有 public API。
- 值类型 → `struct` / `readonly struct`；不可继承类 → `sealed`。
- 热路径方法 → `[MethodImpl(MethodImplOptions.AggressiveInlining)]`。
- 禁用 `IEnumerator` 协程，统一 `UniTask` / `UniTaskVoid`（传 `CancellationToken`）。
- 热路径禁 LINQ / `$"..."`，用 `ZLinq.AsRef()` / `ZString`。
- Burst Job 内禁字符串 / 托管类型 / ZLinq / ZString / UniTask。
- ⚠️ **禁止用 py/ps1 脚本批量修改含中文的 .cs 文件**（用户规则）。

## 10. 常见问题与陷阱（Troubleshooting）

| 现象 | 根因 | 处置 |
|------|------|------|
| 协议导出失败 | `ExporterSettings.json` 仍是 macOS 路径 | 改为 Windows 绝对路径（§4.3） |
| 还原后接口异常 | NuGet 用 `*` 漂移到新版本 | 锁定具体版本（§7） |
| Burst 编译报错 | Job 内用了托管类型 / 字符串 | 改 NativeArray + blittable struct |
| 客户端连不上 | `HotfixHash` 不在服务器白名单 / 签名失败 | 检查构建机签名与白名单（构建与部署 §签名） |
| 对局 desync | Burst 版本不一致 / Hotfix 在 Battle 热更 | 对局前 `BurstVersion` 校验；Battle 禁热更（架构 §4.5.2） |
| 资源不匹配拒进对局 | `AssetManifestHash` 不一致 | 大厅先 YooAsset 热更对齐（架构 §4.5.3） |
| 中文注释乱码 | 用脚本批量改 .cs | 禁止；手动改或专用工具 |
| `HybridCLRData/` 被提交 | force add | `.gitignore` 已排除，撤销提交 |
