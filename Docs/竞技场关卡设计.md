## 0. 设计目标与约束对齐

### 0.1 一句话定位（落到关卡）
> **把"出界即死"做成全场唯一死因，让每一寸地形都既是掩体、又是武器、也是潜在杀机——玩家用落沙材料改造战场，把对手挤/推/炸出竞技场边界即获胜（百战天虫规则）。**

### 0.2 与 ADR / GDD 的硬约束对齐表

| 决策 | 结论（不可动摇） | 对关卡设计的要求 |
|------|----------------|------------------|
| **D1 边界死亡** | `Kill` 为全模式默认；Bounce/Wrap 仅自建房可选 | 地形必须是"悬浮岛 + 四周虚空"，让"被推出/跌出"成为唯一死法；边缘是技术点（K 表：仅边缘 15–55px 内被命中才出界） |
| **D2 缩圈** | MVP 不做缩圈 | 固定硬边界；依靠**地形边缘 + 残局人数收敛**制造空间压缩，而非移动圈 |
| **D4 HP 死亡** | MVP 纯出界即死，无 HP；环境危害=控制/逼走位 | 所有材料布局以"制造击退 / 封路 / 逼走位"为目的，绝不设计致死陷阱；生物只推不杀 |
| GDD §8 | Width/Height ∈[64,1024] 默认 256；Seed 确定性；MaxSpeed 反作弊 | 尺寸与刷新全部 Seed 推导；见 §1/§6 |
| GDD §7 | 道具 Telegraph 90 帧（3s）预告，所有人可见 | 道具锚点 + 刷新时间线确定性，见 §4.1 |

### 0.3 设计支柱 → 关卡落地

| 体验支柱（GDD §2） | 关卡侧落地手段 |
|-------------------|---------------|
| 环境即武器 | 矿物/材料按"反应对"就近布置（木上压水、岩旁熔岩、酸蚀非石），鼓励连招 |
| 低门槛高上限 | 对称出身 + 平坦出生台 + 可拾取道具，新手能走位即上手；地形改造（挖坑/造墙/灌水）提供深度 |
| 短局快节奏 | 尺寸收敛（4 人 256）、刷新密集、残局靠边缘博弈收口，单局 < 8min |
| 可读的混乱 | 地形高度受限保证视线；反应表固定（GDD §5.2）；Telegraph 光柱统一预告 |
| 公平确定性 | 全 Seed 推导、对称布局、最小间距 ≥ Width/4、无运行时随机（§6） |

## 1. 竞技场参数落地（对齐 GDD §8 + 架构 §4.4）

### 1.1 参数总表（权威值与默认值）

| 参数 | 类型 | 范围 | 默认 | 来源 / 说明 |
|------|------|------|------|------------|
| `Width` | int | [64, 1024] | **256** | GDD §8 / 架构 §4.4 `ArenaConfig.Width` |
| `Height` | int | [64, 1024] | **256** | 同上 `ArenaConfig.Height`（MVP 取正方形，便于对称；非正方见 §5） |
| `Seed` | int | 服务器下发 | — | 确定性地形/出生/刷新唯一熵源；见 §6 |
| `BoundaryMode` | enum | Kill/Bounce/Wrap | **Kill** | ADR D1；Bounce/Wrap 仅自建房可选变体 |
| `MaxSpeed` | Fix64 px/s | [280, 360] | **320** | NUMERIC §2 `MaxSpeed(cap)`；反作弊位移上限，每步容忍 `×1.5≈16px` |

> 注：`ArenaConfig` 结构体（架构 §4.4）原样沿用，**未改动**。`MaxSpeed` 不在 GDD §8 表内给默认值，本文依 NUMERIC §2 收口为 **320 px/s**（反作弊位移上限，非玩家移速；玩家 `MoveSpeed`=75、Dash≈210 均远低于此，见 §1.3）。

### 1.2 尺寸–人数推荐映射（模板选择基线）

| 房间人数 N（RoomConfig.PlayerCount） | 推荐 Width×Height | 备注 |
|------------------------------------|------------------|------|
| 2（Duel） | 192×192（或 160×256 纵向） | 1v1，边缘博弈为主 |
| 4（默认乱斗） | **256×256** | P0 默认；残局空间足够 |
| 8（Grand Brawl） | 384×384（或 512×512） | 需更大中心争抢区与更多材料 |

- 下限 64 仅用于极端自建房；规则上合法但体验差（出生间距难满足），不建议 Quick/Ranked 使用。
- 上限 1024 为引擎/带宽边界（环境层快照成本），仅 8 人 BR 远景考虑。

### 1.3 与反作弊 / 物理的一致性校验

- 单步合法位移上限 = `MaxSpeed × dt × 1.5 = 320/30×1.5 ≈ 16 px/步`（NUMERIC §2）。
- 本作所有位移源均在此内：
  - 玩家移动 75 px/s → 2.5 px/步
  - Dash 34px/0.16s ≈ 210 px/s → 7 px/步（ADR §3 / NUMERIC §2.1）
  - 击退爆发 ≤ 5 px/步（K=150 → 50px 总位移摊到 ~10 步，NUMERIC §3.1）
- **关卡侧承诺**：地形不制造任何"加速滑道"使合法位移突破 16px/步（冰面滑行也以 Fix64 速度封顶，不引入额外重力/速度场）。冰面仅改变"阻滑手感"，不增速度上限。

## 2. 出生点规范（确定性 + 最小间距 ≥ Width/4）

### 2.1 生成算法（纯函数 of Seed）

```
输入: ArenaConfig{Width,Height,Seed}, N (=RoomConfig.PlayerCount ∈ [2,8])
派生: spawnRng = Xorshift128Plus(Seed ^ 0x5AAA4E72)   // 独立流，避免与地形/刷新串扰（合法十六进制种子混淆常量）
1. 计算出生环半径 R:
     R = clamp( min(Width,Height) * 0.34, Width/4 * 1.5, min(Width,Height)*0.42 )
     // 保证弦长 ≥ Width/4 且环在边界内（留 16px 安全边距）
2. 基准角 phase = spawnRng.Next() % 360   // 仅旋转对称布局，不改变间距
3. for i in 0..N-1:
     θ = phase + i * (360/N)
     x_i = round( Width/2  + R*cos(θ) )
     y_i = round( Height/2 + R*sin(θ) )
4. 落地修正: 将 (x_i, y_i) 吸附到该列地形表面之上的"出生台"（见 2.2），保证不在虚空/液体中出生
5. 间距校验: 任意两点欧氏距离 ≥ Width/4  → 否则放大 R 重算（确定性，不引入随机）
```

### 2.2 出生台（Spawn Shelf）—— 安全与对称

- 每个出生点下方确定性地"雕刻"一块 **宽 16px、高 4px 的平整 Stone 台**（固体、酸不可蚀、无液体、无虚空），玩家盒（4×6，NUMERIC §2）稳定落于台上，绝不会被埋/淹/跌。
- 出生台所在列的顶部 2px 覆 Sand（粉末，可被挖动但出生瞬间安全），便于玩家开局即获得可改造材料。
- 对称保证：N 个出生台按角度均分，几何对称；地形高度因噪声不同，但出生台是"强制平整"的，故**落点公平性由出生台保证，不依赖地形噪声**。

### 2.3 最小间距证明（对齐 GDD §8 / NUMERIC §9.4）

- 环上相邻两点弦长 `d = 2R·sin(π/N)`。
- 由 2.1 的 R 下界 `R ≥ 1.5·Width/4`，得 `d ≥ 3·(Width/4)·sin(π/N)`。
- 最严苛 N=8：`sin(π/8)=0.3827` → `d ≥ 3·(W/4)·0.3827 = 1.148·(W/4) > W/4` ✓。
- 各档实测（默认参数）：

| N | 尺寸 | R | 弦长 d | 要求 W/4 | 余量 |
|---|------|---|--------|---------|------|
| 2 | 192 | 64 | 128 | 48 | 2.7× |
| 4 | 256 | 88 | 124 | 64 | 1.9× |
| 8 | 384 | 140 | 107 | 96 | 1.1× |

> 全部满足 ≥ Width/4；8 人档余量最小但仍达标，且 R 已留边界安全边距（spawn X∈[52,332]⊂[16,368]）。

## 3. 地形生成规范（heightMap 基底 + 洞穴 + 矿物；对落沙物理友好）

### 3.1 总体形态：悬浮岛 + 四周虚空（支撑 D1 Kill）

- **岛掩码 islandMask(x)**：以中心为圆心的椭圆（半径由尺寸定），边缘用低频噪声微扰出有机轮廓；掩码内=地形，掩码外=**虚空（air）**。
- **虚空即死区**：掩码外整列空气，玩家一旦跌出岛缘 → y 持续增大 → 越过 `y > Height+16` 触发 `PlayerDeath`（GDD §4.2 / 架构 §4.4 死亡判定）。左右越界同理（`x ∉ [-16, Width+16]`）。
- 这把"被推出/跌出"做成唯一死法，**原生满足 D1**，无需额外陷阱。

### 3.2 heightMap 与分层（对齐 GDD §8 "基底+洞穴+矿物"）

```
噪声源(全部 Seed 派生, Fix64, 无浮点发散):
  H_noiser = fbm(x; seedTerrain, oct=4, freq=1/48)      // 主地表起伏
  C_noiser = fbm(x*1.7+13; seedTerrain^0xC, oct=3)      // 洞穴分布
  M_noiser = hashGrid(x,y; seedTerrain^0xA)             // 矿物静脉

逐列 x (islandMask(x)==true):
  surfaceH(x) = floor( Height*0.52 + Height*0.14 * H_noiser(x) )   // 地表线，约 52% 高度处起伏
  for y in [surfaceH(x) .. Height]:
      if 处于"顶部 Sand 带"(y < surfaceH+8): 置 Sand(粉末)
      else if 处于"洞穴带"且 C_noiser 超阈 且 距出生台>20px: 置 Air(挖空)
      else: 置 Stone(固体, 酸不可蚀)
  矿物静脉: 在 Stone 带按 M_noiser 以低概率嵌入 Wood/Water/Lava/Ice 团块(见 3.3)
逐列 x (islandMask(x)==false):
  整列 Air(虚空)
```

- **基底 Base**：Stone 固体，构成岛体结构；酸不可蚀 → 永久结构（酸只能控场非石物质，对齐 GDD §5.2 / SPELL §5）。
- **顶部 Sand 带**：粉末，可被挖动/掩埋/填坑（"环境即武器"的低门槛入口）。
- **洞穴 Caves**：Air 空腔，制造视线遮挡与埋伏点，但**远离出生台与岛缘**（避免开局即坠崖）。

### 3.3 矿物 / 材料布局（对落沙物理友好 + 反应对就近）

> 密度/状态依据 NUMERIC §7.1（Density：Stone255 > Wood240 > Ice220 > Ash170 > Sand180 > Lava130 > Acid110 > Water100 > Poison60 > Fire15 > Steam25 > Smoke20）。布局遵循"重在下、轻在上、反应对相邻"。

| 材料 | 放置位置 | 设计意图（引 法术系统设计.md 反应） |
|------|---------|-------------------------------------------|
| **Sand（顶带）** | 全岛顶部 8px | 可挖/掩埋/填坑；最轻粉末，落沙友好 |
| **Stone** | 岛体主体 | 永久结构；酸不可蚀；熔岩+水→石（瞬墙）的"安全骨架" |
| **Wood 团（可燃）** | 地表下 2–4px、成簇 | 火+木→灰+火蔓延（逼迫走位）；布置在"平台"下，烧穿即塌 |
| **Water 囊（密封）** | Stone 包裹的小腔 | 打破即灌坑；水+火→蒸汽（烟雾遮蔽）；水+毒→稀释；水+熔岩→石墙 |
| **Lava 囊（稀有，低位）** | 岛体下部、近 Stone | 熔岩+木→火推进；熔岩+水→石+蒸汽（瞬墙封路）；高害但可控 |
| **Ice 小块** | 偶发地表 | 冰+火→水（造滑面）；遇火融水 |
| **Acid 囊（P1 稀少）** | 非 Stone 团块外缘 | 酸溶非石（清场/破防），但遇 Stone 无效 → 强调 Stone 结构价值 |

**反应对就近原则（设计师刻意安排的"连招热区"）**：
1. **木上压水**：Wood 平台正下方埋 Water 囊 → 烧木塌落即灌水→蒸汽幕/填坑。
2. **岩旁熔岩**：Lava 囊紧邻 Stone 柱 → 引水即瞬造石墙封路（熔岩+水→石+蒸汽）。
3. **酸蚀非石带**：Acid 囊布置在 Wood/Sand 区边缘 → 酸蚀清场但被 Stone 止步，形成"安全岛"。

### 3.4 视线与可读性（"可读的混乱"）

- **高度上限**：岛体最高不超过 `Height*0.70`，保证相机（clamp 到竞技场）内视线不被单体结构完全遮挡； tall 结构（Stone 柱）宽度 ≤ 12px，避免"盲墙"。
- **边界红光**：岛缘虚空区渲染红色危险光（GDD §4.2 / ADR §1），作为无感引导提示"此处即死"。
- **对称但非雷同**：左右镜像对称保证公平，但用噪声让地形不机械重复，保留探索感。

## 4. 体验节奏（道具 / 生物 / 战斗张力曲线）

### 4.1 道具刷新（Telegraph 90 帧，对齐 GDD §7 + 架构 §7.4）

- **预告机制**：服务器 `SceneScheduler` 在 `TargetFrame` 前 90 帧（3s@30Hz）下发加密 `G2C_SceneEvent`，客户端在 `TargetFrame−90` 播放高对比光柱（RGB565 固定色，对齐 SPELL §5 Telegraph 规范），3s 后落地可拾取，**所有人可见 → 公平争夺**。
- **刷新锚点（确定性，见 §6）**：从 Seed 衍生一组 ~12 个候选锚点（中央高台、各岛瓣顶、矿物热区旁），`SceneScheduler` 每轮从锚点池 + `ItemSpawnTable` 权重选点选类。
- **刷新时间线（确定性预滚，见 §6）**：
  - 首个道具 `TargetFrame = 150`（开局 5s，玩家完成定向）。
  - 间隔 `rand∈[240,420]` 帧（8–14s）一刷；场上同时存活上限：4 人档 ≤3、8 人档 ≤5。
  - 道具超时 5min 未拾取自动消失（架构 §7.4.2）。
- **道具类型与权重（GDD §7 / NUMERIC §8）**：

| 类型 | 权重 | 4 人档效果 | 备注（对齐 D4） |
|------|------|-----------|----------------|
| 武器 | 30 | +30% 弹伤 / +20px 射程，15s | 攻击向 |
| 法术卷轴 | 20 | +1 法术槽（≤4） | 引 SPELL §8：卷轴进 `ItemSpawnTable` |
| 治疗药水 | 25 | **纯 Kill 模式=净化瓶：清除减速/致盲等环境 debuff + 1s 免环境控制（不免疫击退）** | NUMERIC §8 / GDD §7：no-HP 兜底=净化瓶（不回血、不免疫击退） |
| 护盾 | 10 | 3s 免疫环境 DoT+直伤，**不免疫击退** | ADR D4 / NUMERIC §6.5：否则破 Kill 张力 |
| 速度增益 | 10 | +50% 移速，5s | 机动 |
| 炸弹 | 5 | =Bomb 法术 1 发（击退 150） | 主推人向量 |

> **D4 一致性铁律**：任何道具/法术在 no-HP 模式下**不得提供"免疫击退"**（护盾仅免 DoT/直伤）。这写入关卡刷新配置，避免破坏 Kill 招牌（NUMERIC §9.7 / §11）。

### 4.2 生物刷新（对齐 生物AI规格.md）

- **MVP 最小集**：每竞技场 1 中立（Slime）+ 1 敌对（Firebat），位置由 Seed 确定性生成。
- **布置策略**：
  - **Firebat（敌对，重弹级击退≈90/30px）**：置于**中央争抢高台**附近，作为第三方推力威胁，天然制造"抢中心=冒被推风险"的张力；不直接致死（D4），只推。
  - **Slime（中立，轻弹级击退≈45/15px）**：置于**外围岛瓣**，可被推挤/利用（吞 Sand/Water 微调地形），轻度混乱。
- **确定性**：生物 PRNG 流 `seed = ArenaConfig.Seed ⊕ entityId`（CREATURE §2.1），走玩家层 GGPO 回滚（CREATURE §2.4）。
- **致死一致（D4）**：生物攻击=击退/控制，死亡只由出界判定；其击退同样受"边缘 15–55px 才出界"铁律约束（CREATURE §7）。
- **P0 不刷新 / 不重生**；P1 扩展：多生物种类 + 群聚 + 周期性重生（在 Seed 时间线内确定性排布）。

### 4.3 战斗张力曲线（4 人默认档）

```
威胁/张力
 高 │                                    ╱─────╲        ╱─╲
    │          ╱─╲        ╱─╲    ╱─╲   ╱       ╲──────╱   ╲___ (残局 1v1 边缘博弈)
    │   ╱─╲   ╱   ╲  ╱─╲╱   ╲  ╱   ╲ ╱                             
    │  ╱   ╲ ╱     ╲╱     ╲╱     ╱╱    (道具 Telegraph 每 8–14s 尖峰)
 低 │_╱     ╲__________________________________________________________
    └──┬─────┬─────────┬──────────────┬─────────────── 时间
      0    30s       60s           180s             ~<8min
     定向  首杀窗口   材料战中期      残局收敛
```

| 阶段 | 时间窗（4 人档） | 张力来源 | 设计目标（NUMERIC §10） |
|------|----------------|---------|------------------------|
| **P1 定向** | 0–~30s | 散开、抢首道具；无致死压力 | 玩家完成落地与定向 |
| **P2 首杀窗口** | ~30–60s | 首道具争夺 + 早期材料连招 | **首杀 30–60s**（无瞬杀） |
| **P3 材料战** | ~60–180s | 道具密集（每 8–14s）、生物骚扰、多杀 | 中期击杀间隔 20–40s |
| **P4 残局收敛** | 180s–结束 | 人数低、地形开阔、边缘博弈 | 1v1 边界博弈 30–90s/局；总时长 <8min |

- **尖峰来源**：每次道具 Telegraph（90 帧光柱）制造一次"争夺聚集"张力尖峰；生物（尤其 Firebat 中心游荡）持续提供基线扰动。
- **曲线保证**：通过"尺寸收敛（256）+ 刷新密集 + 边缘即死"三件套，使单局自然收敛到 <8min，无需缩圈（D2）。

## 6. 确定性约束（地形 / 出生 / 刷新全由 Seed 推导，无运行时随机）

### 6.1 熵源唯一化

- 唯一熵源 = `ArenaConfig.Seed`（服务器下发，对齐 GDD §8 / 架构 §4.4）。
- 所有"随机"均为**确定性 PRNG 流** `Xorshift128Plus`（Burst 兼容、跨平台一致，架构 §8.2 / GAMEPLAY §4.1.4），**绝不使用** `UnityEngine.Random` / `Time.deltaTime` / 浮点抖动。
- 分流域避免串扰（CREATURE §2.1 同思路）：

| 流 | 种子派生 | 用途 |
|----|---------|------|
| terrainRng | `Seed ^ 0x7EAA51C3` | heightMap / 洞穴 / 矿物 |
| spawnRng | `Seed ^ 0x5AAA4E72` | 出生环相位 / 台位置（§2.1） |
| itemRng | `Seed ^ 0x173E5A5F` | 道具锚点选择 / 类型 / 间隔 |
| creatureRng | `Seed ^ entityId`（每实体） | 生物游荡/感知（CREATURE §2.1） |

> **种子混淆常量约束**：上表 `0x…` 须为**合法十六进制字面量**（仅 `0–9`/`A–F`，不含 `P`/`M` 等非法字符、不含 `'` 撇号分隔符），否则照抄进 `Xorshift128Plus(ulong seed)` 会编译失败。以上常量已修正为合法 32 位 hex（对齐 PRNG 规范）。

### 6.2 三大确定性保证

1. **地形确定性**：`heightMap` + `islandMask` + 矿物团块全部由 `terrainRng` 推导；客户端在 Battle 加载阶段一次性接收静态层（架构 §1.1 静态层），三平台哈希一致。
2. **出生确定性**：`spawnRng` 推导环相位与半径（§2.1），出生台强制平整 → 落点对称且安全，无运行时随机。
3. **刷新确定性（时间线预滚）**：`SceneScheduler` 在**对局开始（Frame=0）即由 `itemRng` 预滚整局道具/生物刷新时间线**（目标帧 + 锚点 + 类型全部确定）。
   - 该时间线是 `(Seed, ArenaConfig, N)` 的纯函数 → 服务器与任何客户端重放完全一致。
   - 事件仍经 `G2C_SceneEvent` 帧延迟加密下发（架构 §6.3/§6.5），但**内容本身由 Seed 决定**，无运行时熵。
   - 此设计同时满足：帧同步回滚友好（事件为外部注入、不进玩家层回滚）、反作弊（客户端无法提前预测明文，但重算时间线必与服务器一致）、确定性验收（架构 §12.1）。

### 6.3 禁止项（写入实现约束）

- ❌ 任何 `System.Random` / `UnityEngine.Random` / `Guid` / 时间戳作为地形/出生/刷新输入。
- ❌ 客户端本地计算地形或刷新位置（仅服务器 `WorldBuilder` + `SceneScheduler` 依据 Seed 生成，客户端消费）。
- ❌ 运行时浮点噪声（必须 Fix64 + 整数化高度）。
- ✅ 所有 RNG 状态进确定性快照（生物走玩家层回滚；地形/刷新为静态/预滚，不参与逐帧回滚）。
