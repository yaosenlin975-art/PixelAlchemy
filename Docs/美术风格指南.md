# NoitaCA 美术风格指南（Art Style Guide / Art Bible）

> **文档版本**：v1.0
> **创建日期**：2026-07-06
> **状态**：草稿 / 待评审
> **归属学科**：美术（Art）
> **维护人**：许清楚（产品经理）+ 美术
> **强约束声明**：本游戏是**像素落沙确定性渲染**，所有像素颜色必须可编码进 `PixelData.Color`（`ushort` 打包 **RGB565**，16 位）。美术表达必须在 16 位色域内，且**不能依赖任何后处理随机 / 每帧随机抖色**，否则破坏帧同步确定性（详见 §2）。

## 1. 视觉基调（Visual Tone）

| 维度 | 取向 |
|------|------|

## 2. 确定性渲染约束（必读）

对齐 `玩法重构方案.md §4.1.1`：

```csharp
public struct PixelData : IComponentData
{
    // ...
    public ushort Color;   // 打包 RGB565, 缓存效率
    // ...
}
// Total 16 字节/像素
```

### 2.1 RGB565 色域约束

- 颜色格式：**R5 G6 B5**（共 16 bit，`ushort`）。
- 每通道精度：R/B 32 级、G 64 级。
- **所有材质 / 像素颜色必须定义为 16 位 RGB565 常量**，与 `MaterialDefinition.DefaultColor`（`ushort`）一致。

### 2.2 禁止项（破坏确定性）

| 禁止 | 原因 |
|------|------|
| 后处理随机抖色 / 噪声扰动像素色 | 每客户端随机不同 → desync |
| 每帧基于 `Time` 的色相偏移 | 非确定性（架构文档 §8.2 禁用 `Time.deltaTime`） |
| 浮点后处理调色（bloom 依赖浮点） | 跨平台浮点发散 |
| 超出 16 位的 HDR / 32 位色 | 无法打包进 `ushort Color` |

### 2.3 允许项

- 固定的 16 位调色板（见 §3）。
- Telegraph 光柱用**确定性程序化动画**（基于整数帧计数，非浮点），见 §5。
- 周期调和的 100ms 视觉插值（架构文档 §4.2.3 / §8.3）使用确定性线性插值。

## 3. 主调色板（12 材料 = RGB565）

> 与 GDD §5.1 材料一一对应。颜色为权威 `DefaultColor`，美术资源（特效 / 图标）须从本板取色，保证热更前后视觉一致（避免视觉 desync，架构文档 §4.5.3）。

| # | 材质 | RGB565 (hex) | R,G,B (0-31/0-63/0-31) | 视觉描述 |
|---|------|------|------|------|
| 1 | 沙 Sand | `0xFF90` | 31,52,16 | 暖黄沙色 |
| 2 | 水 Water | `0x029F` | 0,20,31 | 深蓝透明感 |
| 3 | 石 Stone | `0xA294` | 20,20,20 | 中性灰 |
| 4 | 木 Wood | `0xE648` | 28,18,8 | 棕褐 |
| 5 | 火 Fire | `0xF900` | 31,8,0 | 亮橙红 |
| 6 | 烟 Smoke | `0x9264` | 18,18,18 | 浅灰 |
| 7 | 蒸汽 Steam | `0xB3DF` | 22,30,31 | 近白青 |
| 8 | 冰 Ice | `0x73FF` | 14,31,31 | 青白 |
| 9 | 熔岩 Lava | `0xF9E0` | 31,15,0 | 炽橙 |
| 10 | 酸 Acid | `0x7BE0` | 15,31,0 | 荧光绿 |

## 4. 材料视觉表现规范

| 材质 | 形态动效 | 视觉约定 |
|------|---------|---------|
| 沙 | 颗粒下落堆积 | 单色块、无渐变 |
| 水 | 流动、表面微光（确定性） | 边缘略亮，内部均匀 |
| 石 | 静止 | 哑光、无高光 |
| 木 | 静止 | 纹理用 1-2 像素暗线暗示 |
| 火 | 短寿命闪烁（基于帧计数） | 中心亮、边缘暗，不随机 |
| 烟 | 上升消散 | 半透明感用降低亮度模拟 |
| 蒸汽 | 上升冷凝 | 近白、柔边 |
| 冰 | 阻滑反光 | 高亮边 |
| 熔岩 | 流动发光 | 最亮橙，暗示高温 |
| 酸 | 流动腐蚀 | 荧光绿，边缘脉动（帧驱动） |
| 毒 | 挥发 | 毒绿，气泡用 1px 亮点 |
| 灰烬 | 堆积 | 暗灰，轻飘感 |

**统一规则**：每种材料**单一主色 + 至多 1 个明暗变体**，变体必须预定义（非随机），保证所有客户端一致。

## 5. Telegraph 光柱特效风格

对齐架构文档 §7.4.3（TargetFrame - 90 帧开始播放，3 秒预告）：

| 属性 | 规范 |
|------|------|
| 形状 | 垂直光柱 + 落点圆环 |
| 颜色 | 使用调色板中"警示"色（建议 `0xF900` 火橙或 `0x7BE0` 酸绿），不与材料混淆 |
| 动画 | **整数帧驱动**（0→90 帧由暗到亮），禁用 `Time.deltaTime` |
| 透明度 | 阶梯式（每 15 帧一档），确定性 |
| 目的 | 让所有玩家提前 3 秒看到"道具将在此落下"，公平争夺 |

## 6. 玩家 / 法术 / 生物 / 装备图标风格

| 元素 | 风格 |
|------|------|
| 玩家 | 简洁像素小人（4-8 色），颜色区分房主 / 排名 |
| 法术图标 | 几何符号（火=三角、冰=六边形、毒=水滴），单色 + 描边 |
| 生物 | 圆润像素块，眼睛提示 AI 状态（Idle/Chase…） |
| 装备 / 道具 | 扁平图标，1-2 色，Telegraph 落点处显示同款图标 |

**约束**：图标资源进 YooAsset，热更须版本对齐（架构文档 §4.5.3），否则部分客户端看不到图标 → 不公平。

## 7. UI 美术基调

| 维度 | 规范 |
|------|------|
| 字体 | 像素字体（中文用点阵像素中文字体） |
| 配色 | 沿用 §3 调色板，UI 主色取石/冰中性色，强调色取火橙 |
| 状态色 | 危险=火橙、安全=冰青、禁用=暗灰（与材料语义一致） |
| 布局 | 信息密度高但分区清晰：小地图 / 生命 / 道具槽 / 快捷表情 |
| 确定性 | UI 不依赖浮点动画；过渡用整数帧 / UniTask 编排 |

---

## 8. 美术资源与确定性 checklist

- [ ] 所有像素颜色为 16 位 RGB565，定义于 `MaterialDefinition.DefaultColor`
- [ ] 无后处理随机抖色 / 浮点调色
- [ ] Telegraph / 动效基于整数帧计数，非 `Time.deltaTime`
- [ ] 材料变体预定义，非运行时随机
- [ ] 图标 / 特效资源纳入 YooAsset，热更版本对齐
- [ ] 主调色板（§3）为唯一取色源，美术资产不得引入板外色
