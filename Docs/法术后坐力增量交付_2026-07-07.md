# NoitaCA 法术后坐力设计增量交付（2026-07-07）

## 一、增量背景
用户诉求：添加"不同法术发射时不同后坐力"设定。主理人（程驭）走正式团队协作流程——建团队 `noita-recoil-v2`，并行派发谋远（玩法机制）/舒衡（数值表）/端木林（确定性模拟落地），最终汇编落盘四份文档 + 刷新 ADR D6 决策。

## 二、核心决策（ADR D6，主理人拍板）
- **速度冲量模型**：施法瞬间施法者沿 `AimAngle` 反向获得速度冲量（非固定位移），受落沙阻力/边界反弹影响。
- **允许自爆**：后坐力可把施法者推出界即死（高风险战术，保留）。
- **适用范围**：仅发射类 8 法术有后坐力；StoneWall/Lava（非发射/喷发类）后坐力=0（谋远裁定，SPELL §3.4.7 权威分类）。
- **确定性红线**：后坐力 = f(SelectedSpellSlot 派生法术, AimAngle)，纯模拟派生，**不进 InputPayload、必进状态哈希**，可 GGPO rollback 重放。
- **协议零变更**：InputPayload 8B 不变、OuterMessage.proto 不变、ProtocolVersion 不变（端木林确认）。

## 三、主理人拍板的两处冲突（文档 §6 卡点）
- **A·阻力系数**：物理层权威 `DRAG=0.90/步`（非 brief 的 0.85）。自由空间位移 `D=v0·0.3333`。自爆护栏阈值 `T` 由 90px 重算为 **135px（2.25×bodyWidth=60）**，`RecoilSpeed_max=405` 作护栏设计封顶。
- **B·StoneWall/Lava 分类**：采用 `SPELL §3.4.7` 权威分类，StoneWall（地形生成）/Lava（喷发）非发射类→后坐力=0；重型有后坐力法术收敛为 Heavy/Bomb/Fire。

## 四、数值表（舒衡 NUMERIC §13 v0.6）
`RecoilSpeed`(px/s)：Bolt120 / Water160 / Ice220 / Poison240 / Acid260 / Heavy320 / Bomb360 / **Fire390（发射类封顶）**；Shield·Blink·StoneWall·Lava = 0。
`DRAG=0.90/步`；安全阈值 `T=N=135px`；`V_MAX=1000px/s`。

> A/B 字面矛盾消解：护栏仅受发射类反冲约束，现发射类封顶=Fire 390（位移 130px < T=135px），故 T=135 对所有现版本法术成立且略保守；`RecoilSpeed_MAX_PHYS=405` 仅留作护栏设计封顶（包络未来 >390 新法术），不对应任何现值。

## 六、一致性核验矩阵
- ADR D1–D6 全绿：出界即死 / 不缩圈 / Dash 无 i-frames / 无 HP / 固定力度+循环切法术 / 后坐力速度冲量模型。
- 不破 §9.7 免推铁律：后坐力是"自我推力"≠"免推"（仍可被外界击退/推动叠加）。
- 不破 D3 无 i-frames：后坐力纯位移、不免疫任何效果；与 Dash 可共存叠加互不影响。
- 协议零变更：InputPayload 8B（MoveX/Y+ActionFlags+AimAngle）、ProtocolVersion=2、无新字段、不 bump 版本。
- 确定性：后坐力仅修改 VelX/VelY + 位置，`MurmurHash3` 状态哈希天然覆盖，无新增哈希字段、回滚零成本。
