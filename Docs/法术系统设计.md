## 1. 概述与目标
- 法术是"环境即武器"的延伸：用元素改造战场、逼走位、造墙/封路、把对手推/挤出边界。
- MVP 原则（D5 定稿 + 数值对齐）：每法术**独立冷却 + 共享魔力双闸**；**固定力度**（无 AimPower 输入字段，预留 uint8）；循环切换 + 单施法。

## 2. 法术分类
| 类别 | 定位 | 代表 |
|------|------|------|
| 攻击 Attack（物理投射） | 直接命中 + 击退 | Bolt（轻弹）/ Heavy（重弹）/ Bomb（爆炸） |
| 元素 Element | 注入元素材料 + 点燃/控制 | Fire / Ice / Poison / Acid |
| 环境改造 Env | 改变落沙地形 | Water（水流术）/ Lava（熔岩喷发）/ StoneWall（石墙术） |
| 位移 Mobility | 自身快速移动 | Blink（闪现） |
| 防御 Defense | 阻挡 / 挡伤 | Shield（护盾泡） |

## 3. 施法流程（确定性）
### 3.1 法术槽与切换（循环切换 + 单施法）
- 4 槽（GDD §6.2）；`SelectedSpellSlot`(0–3) 由 `SelectNext/Prev` **确定性自增/自减**（循环）推导，是 `PlayerData` 字段、进快照、可回滚。
- 切换**免费**（不耗蓝/不耗回合），对齐数值锚点。
- `SpellId` 字段循环模型下恒 = 0；`Spell1/Spell2` 位保留但永不启用（收口 A5）。
### 3.2 施法解析
- `CastSpell`(Attack 边沿) → 读 `slots[SelectedSpellSlot]` → 在玩家位置、方向=`AimAngle`[0,628] 生成投射物/区域。
- 命中以**像素采样**判定；范围效果**写入像素**，参与 GDD §5.2 反应表。
### 3.3 力度模型（MVP 固定）
- 所有法术初速为法术常量（**无 AimPower 字段**）；抛物线类用固定初速 + 重力（Fix64）得确定弧线。
- 蓄力 `AimPower` 延后 P1（需 ProtocolVersion bump，见 input-arch Q5；uint8 预留）。

### 3.4 后坐力机制（Recoil）

> 设计意图：给"发射类法术"加上可感知的反冲代价与风险，强化走位博弈与边缘张力；纯模拟派生，不污染 InputPayload、必进状态哈希。数值（量级 K）由 `数值平衡设计.md §13.1 RecoilSpeed 表`（舒衡，原 `RECOIL_TABLE`）定义，本机制只定玩法规则。

#### 3.4.1 定义
- **后坐力**＝施法者在施法瞬间获得的一个**速度冲量**（velocity impulse，非固定位移），方向沿 `AimAngle` 反向。
- 它是施加到玩家速度向量上的**一次性冲量**，之后与移动速度一样受落沙阻力衰减、墙体碰撞、边界判定影响。
- 后坐力是施法者**自身主动产生**的"自我推力"，**不属于** ADR §9.7 铁律禁止的"免推"（免推＝不被外界推；后坐力＝自我推，仍可被外界击退/推动叠加）。

#### 3.4.2 触发时机与 SelectedSpellSlot 绑定
- 触发：`ActionFlags.Attack` **边沿**（= `CastSpell`）→ 模拟层先解析当前法术 → 施法成功后查 `NUMERIC §13.1 RecoilSpeed 表` → 沿 `AimAngle` 反向叠加速度冲量。
- 绑定关系：后坐力与生成的投射物**共用同一次法术解析**——`spell = slots[SelectedSpellSlot]`。即"放什么法术，就吃该法术的后坐力"，二者由同一推导法术决定，自洽。
- `SelectedSpellSlot` 是 `PlayerData` 字段（由 `SelectNext/Prev` 位确定性推导、进快照、可回滚），任意帧法术唯一确定，后坐力可重放。
- **门控一致**：后坐力仅在施法成功时产生（Mana + 每法术 CD 双闸通过）；被拒（无蓝 / 冷却中）则无投射物亦无后坐力。

#### 3.4.3 方向
- 反冲方向 = `反向(AimAngle)`（即 `AimAngle + π`，量化域 [0,628] 内取补）。
- 例：瞄准右方（0）施法 → 施法者获得向左的反冲；瞄准正上施法 → 获得向下反冲（可被重力/落沙放大位移）。
- 方向仅由 `AimAngle`（已在 `InputPayload`）与当前法术（由 `SelectedSpellSlot` 推导）决定，无新增输入。

#### 3.4.4 与位移系统的关系（三速度来源可叠加）
玩家速度由三种**逻辑来源**组成，物理上叠加进同一速度向量（`PlayerData.VelX/VelY`），**三者可叠加、且都不免疫任何效果**（与 ADR D3 一致）：

| 来源 | 产生 | 性质 | 衰减 |
|------|------|------|------|
| **MoveVelocity** | `MoveX/Y` 输入意图，每帧由模拟施加 | 玩家主动位移 | 无（持续输入，可反向抵消） |
| **KnockbackVelocity** | 外界命中（法术/爆炸/反应）单向冲量 | 外界推 | 按 dragCoef 衰减（见 NUMERIC §3.1） |
| **RecoilVelocity** | 施法者自身发射的单向冲量（本机制） | 自我推 | 同落沙阻力衰减 |

- **叠加规则**：边移动 + 边被击退 + 边后坐，三者在速度向量上相加；任一来源都**不含 i-frames / 免推标志**。
- **实现提示（非代码）**：MVP 推荐将三者冲量统一加总到 `PlayerData` 的单一速度向量（`VelX/VelY`），回滚与状态哈希只关心该向量与位置。若端木林为实现手感需内部分离 `RecoilVelocity` 通道，该通道**必须作为 `PlayerData` 字段进快照、进状态哈希**（确定性红线）。

#### 3.4.5 边界与反弹交互
- **默认 Kill 边界（ADR D1）**：后坐力可把施法者推出边界 → 触发 `PlayerDeath`（自爆 / 同归于尽，高风险战术，**允许**）。Kill 模式下边界**不反弹**。
- **可选 Bounce 边界（房主配置）**：速度撞边界按弹性系数 `restitution`（**< 1**，具体数值由物理/数值层定）反射；玩法预期＝反弹**消耗能量而非无限弹**，避免贴墙无限弹。
- **墙体碰撞**：撞固体（Stone / Wood / Ice）速度分量被阻挡 / 滑移（不穿墙），与 Dash 撞固体即停一致；气体 / 液体 / 粉末不阻挡。
- 反弹 / 碰撞的弹性系数与滑移规则由物理层（端木林）实现，本机制只定"消耗能量、不免疫、可被推出界"的玩法预期。

#### 3.4.6 与 Dash 的关系（不冲突）
- Dash（ADR D3）＝边沿一次性**位移**（无资源、CD 1.8s、无 i-frames）；后坐力＝**速度冲量**。二者机制不同、可共存叠加。
- 若施法帧同时含 `Dash` 位：模拟先执行 Dash 位移、再叠加后坐力速度（或同帧合成），最终位置同时反映两者。
- 二者**都无 i-frames、都不免疫任何效果**（含击退 / 材料控制），与 §9.7 一致；后坐力不取消 Dash，Dash 也不免疫后坐力。

#### 3.4.7 适用范围清单（量级以 `NUMERIC §13.1 RecoilSpeed 表`（舒衡）为准，单位 px/s @30Hz）
| 法术 | 发射类? | 后坐力 | 量级 RecoilSpeed(px/s) | 备注 |
|------|---------|--------|------------------------|------|
| Bolt 轻弹 | 是 | 有 | 120 | 基础弹，反冲最弱 |
| Heavy 重弹 | 是 | 有 | 320 | 主反冲源之一 |
| Bomb 炸弹 | 是（投掷） | 有 | 360 | 抛物投掷反冲 |
| Fire 火球 | 是 | 有 | 390 | 直线弹→火 |
| Ice 冰锥 | 是 | 有 | 220 | 高速弹 |

| Acid 酸弹 | 是 | 有 | 260 | 直线/锥 |

| StoneWall 石墙术 | 否（定点召唤） | 无 | 0 | 地形生成，非自身发射质量，无后坐力（主理人裁定，与 Shield/Blink 同列） |

#### 3.4.8 确定性约束（红线）
- 后坐力 = `f(SelectedSpellSlot 推导法术, AimAngle)`，**纯模拟派生**，**绝不进 `InputPayload`**（8B 结构不变：MoveX/Y + ActionFlags + AimAngle）。
- 可重放：`SelectedSpellSlot` 由 `SelectNext/Prev` 位确定性推导并进入快照，回滚重放 `Attack` 边沿即可重建后坐力，无额外状态。
- **状态哈希**：后坐力仅修改 `VelX/VelY` 与位置（均 `PlayerData` 字段），天然被 `MurmurHash3` 状态哈希覆盖（每 60 帧比对）；不向哈希引入新字段，不破坏 wire 稳定。

#### 3.4.9 自爆平衡护栏（玩法侧需求，数值由舒衡 §13.2 锁定实现）
- **战术张力（允许）**：重型发射类法术（Heavy / Bomb / Fire）**贴边放大招有自爆（出界即死）风险**，是发射类法术的招牌张力，设计为**有意保留**（StoneWall / Lava 裁定无后坐力，不参与该自爆张力；其近身危害为另一类风险）。
- **护栏实现（舒衡 §13.2，主理人 2026-07-07 锁定）**：基于 `DRAG=0.90/步` 的自由空间位移 `D = v0·0.3333`，最坏位移（Fire 390）→ `D≈130px`；安全阈值 **T = 135px（=2.25×bodyWidth，bodyWidth=60 为 [TUNING] 假设，见 NUMERIC §13.4 待复核）**，亦作预警半径 `N=135px`。护栏以"安全半径"实现（非硬位移 clamp），由舒衡在 `数值平衡设计.md §13.2` 落盘；本机制仅标注需求与验收，不给数值。
- **⚠ 与初版 brief 的偏差（待主理人确认）**：初版 brief 曾写"单次自由空间位移 ≤ 90px（≈1.5 身位）"，但按现锁定物理（Fire 390 × 0.3333 ≈ 130px）实际最坏位移已超 90px，故护栏以舒衡 §13.2 的 **T=135px** 为准，90px 不再作为安全半径。若主理人要求"硬 clamp 位移 ≤90px"，须作为额外硬上限叠加（会改变手感、与 DRAG 衰减位移并存），需单独拍板。
- **护栏效果（玩法侧验收）**：① **开阔地**（距边界 > 135px）放任意法术，反冲自由位移 ≤ 130px，**不会被自己推出界**（不误杀）；② **贴边**（距界 ≤ 135px）**朝场内发射**，反冲朝外、位移足以出界，**自爆风险真实保留**（战术张力成立）。即"贴边自爆"是设计有意保留的张力，"平地误杀"是被护栏消除的意外。

## 4. 魔力与冷却资源（MVP 双闸，数值锚点 v1）
- **共享魔力 Mana**：上限 100，回复 15/s（模拟态，非 InputPayload 字段，不破坏 wire 稳定）。
- **每法术独立冷却**（帧@30Hz）：同法术不可连发，受冷却闸。
- **法术耗蓝**（舒衡 NUMERIC 锚点）：
  - 轻弹 Bolt 10｜重弹 Heavy 25｜爆炸 Bomb 38
  - 材料类 Water / Lava / StoneWall 12–20
  - 元素/防御/位移类耗蓝见 NUMERIC 文档（下表标注初值，待收敛）。
- 切换免费；施法消耗仅蓝 + 冷却。

## 5. 法术与落沙物质的相互作用
- 法术生成材料进入 CA 模拟，参与 GDD §5.2 反应表（水+火→蒸汽、火+木→灰烬、熔岩+水→石+蒸汽 等）。
- 反制：火球遇水→蒸汽（灭火）；冰锥遇火→融水；毒弹遇火→爆炸；水流遇熔岩→瞬造石墙；酸蚀除石外物质。
- **可读性（B2 折中）**：大范围/延迟法术（Bomb / Lava / Poison）落地前播 **Telegraph 落点光圈**（高对比固定色 + 形状，RGB565 内）；瞬发直线弹（Bolt / Heavy / Fire / Ice）无 Telegraph（弹道可见）。

## 7. MVP 范围与里程碑
- **MVP（P0，6 个）**：Bolt / Heavy / Bomb / Fire / Water / StoneWall + 移动层 Dash。覆盖攻击/元素/环境三类；防御 Shield 延 P1。
- **P1（6 个）**：Ice / Poison / Acid / Lava / Shield / Blink + 蓄力(AimPower)。

## 8. 接口说明
- 数值（舒衡）：速度/冷却/耗蓝/击退 K 表以 数值平衡设计 为准。
- 协议（端木林）：施法=Attack 位；切换=SelectNext/Prev；SelectedSpellSlot 推导；MVP 无 AimPower 字段；mana 为模拟态不进 InputPayload。
- 美术（柯桥/涂彩）：色落 RGB565；Telegraph 高对比固定色 + 形状。
- 关卡（陆关）：法术卷轴刷新权重进 ItemSpawnTable。
