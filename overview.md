## 关键设计决策
- **确定性优先**：所有影响世界状态的操作必须收敛为 30Hz `InputPayload`；元操作（暂停/表情）不进确定性流。
- **抽象输入集（12 动作）**：Move / Aim / CastSpell / SelectSpellNext / SelectSpellPrev / Jump / Pickup / Drop / UseConsumable / Dash / PauseMenu / QuickChat。前 6 个直连既有协议；后 4 个（Select*/UseConsumable/Dash）为协议扩展；末 2 个不进流。
- **跨平台字节级一致**：Core 层 `NormalizeMove()`(模长≤100) + `Fix64Math.Atan2` 量化(AimAngle [0,628])，整数位运算掩盖硬件/采样差异，三平台 MurmurHash3 一致。
- **协议零破坏**：`InputPayload` wire 结构不变，仅扩展 `ActionFlags` 位定义（bit6–9）。

## 待用户/策划拍板的开放项（输入相关）
1. **Dash** 是否保留为真实机制（架构已预留 bit9，但消耗资源/手感未定）。
2. **抛物线力度 AimPower** 是否需（如投掷类道具），架构已预留扩展位。
3. **法术切换模型**：架构建议「循环切换 + 单施法」，UI 文档提供「QE/滚轮 + 屏幕按钮」两套映射。
